# 電子帳簿保存法（電帳法）要件ガイド

> 税理士・会計事務所向けCRMナレッジ
> 最終更新: 2026-03-23

---

## 概要

電子帳簿保存法は、国税関係帳簿書類の電子的な保存を認める法律。
2024年1月1日から**電子取引データの電子保存が完全義務化**された。

### 3つの保存区分

| 区分 | 対象 | 義務/任意 | 概要 |
|------|------|----------|------|
| 1. 電子帳簿等保存 | 会計ソフト等で作成した帳簿・書類 | **任意** | 自己が作成した帳簿・書類をデータのまま保存 |
| 2. スキャナ保存 | 紙で受領・作成した書類 | **任意** | 紙の書類をスキャンしてデータで保存 |
| 3. 電子取引 | メール・Web等で授受した取引データ | **義務** | 電子的に授受した取引情報をデータのまま保存 |

---

## 1. 電子帳簿等保存（任意）

### 対象

- 会計ソフトで作成した仕訳帳、総勘定元帳、補助簿等の帳簿
- パソコンで作成した請求書・見積書等の控え（自己発行分）

### 保存要件

#### 通常の電子帳簿（最低限の要件）

| 要件 | 内容 |
|------|------|
| システム関係書類の備付け | 操作マニュアル、システムの概要書等を備え付ける |
| 見読可能性の確保 | ディスプレイ・プリンタで速やかに出力できること |
| 検索機能の確保 | 取引年月日、取引金額、取引先で検索できること |

#### 優良な電子帳簿（追加要件あり）

通常の要件に加え、以下を満たす帳簿。優良な電子帳簿として届出した場合、**過少申告加算税が5%軽減**される。

| 追加要件 | 内容 |
|---------|------|
| 訂正・削除の履歴の確保 | 帳簿の記録事項について訂正・削除を行った場合、その内容を確認できること |
| 相互関連性の確保 | 帳簿の記録事項と関連する他の帳簿の記録事項との間で相互にその関連性を確認できること |
| 検索機能の強化 | 範囲指定検索、2以上の項目の組合せ検索が可能であること |

#### 優良な電子帳簿のメリット

| メリット | 内容 |
|---------|------|
| 過少申告加算税の軽減 | 申告漏れがあった場合の過少申告加算税が**5%軽減**（10%→5%、15%→10%） |
| 適用条件 | 所轄税務署に「国税関係帳簿の電磁的記録等による保存等に係る過少申告加算税の特例の適用を受ける旨の届出書」を提出 |

## 2. スキャナ保存（任意）

### 対象書類

| 区分 | 書類例 | 入力期間 |
|------|--------|---------|
| 重要書類 | 契約書、請求書、領収書、納品書等 | 最長約2ヶ月+7営業日以内 |
| 一般書類 | 見積書、注文書、検収書等 | 適時に入力（期間制限なし） |

### 保存要件

| 要件 | 内容 | 詳細 |
|------|------|------|
| 入力期間の制限 | 受領後、最長約2ヶ月+7営業日以内に入力 | 「業務サイクル方式」の場合。「早期入力方式」では受領後おおむね7営業日以内 |
| 解像度要件 | 200dpi以上 | A4サイズの書類で約387万画素以上 |
| カラー画像 | 赤・緑・青それぞれ256階調以上（約1,677万色以上） | 一般書類はグレースケール可 |
| タイムスタンプ | 入力期間内に一般財団法人日本データ通信協会認定のタイムスタンプを付与 | 訂正削除の履歴が残るシステムの場合は不要 |
| 読取情報の保存 | 解像度、階調、大きさの情報を保存 | 受領者が読み取る場合 |
| バージョン管理 | 訂正又は削除の事実及び内容を確認できること | - |
| 入力者情報 | スキャナで読み取った者又はその者を直接監督する者の情報を確認できること | - |
| 帳簿との相互関連性 | 帳簿の記録事項と関連付けられること | - |
| 見読可能性 | 14インチ以上のカラーディスプレイ（解像度1024×768以上）で、4ポイントの文字が認識できること | - |
| 検索機能 | 下記「検索要件」参照 | - |
| システム関係書類の備付け | 操作マニュアル等 | - |

### 2024年1月以降の緩和事項

| 緩和内容 | 詳細 |
|---------|------|
| 相互牽制要件の廃止 | 以前は必要だった相互けん制（入力者と確認者の分離）が不要に |
| 定期検査要件の廃止 | 以前は必要だった定期的な原本との突合せが不要に |
| 原本の即時廃棄可 | スキャン後、入力期間内にスキャンしていれば原本を即時廃棄可能 |

## 3. 電子取引（義務）

### 対象取引

電子的に授受した取引情報に係るデータ。2024年1月1日以降、電子データでの保存が**義務**。

| 電子取引の例 | 具体例 |
|-------------|--------|
| メール添付 | PDF請求書、Excel見積書等のメール添付での授受 |
| Webダウンロード | ECサイトでの購入時の領収書ダウンロード |
| クラウドサービス | クラウド請求書サービス（freee、マネーフォワード等）からの受領 |
| EDI取引 | 電子データ交換による取引 |
| ペーパーレスFAX | 複合機でPDF保存されるFAX |
| SNS・チャット | LINE、Slack等での取引情報のやり取り |
| クレジットカード明細 | Web上で確認する利用明細 |
| 交通系ICカード | Webで取得する利用明細 |

### 保存要件

以下のいずれかの措置を講じた上で、検索要件を満たして保存する。

| 措置（いずれか1つ） | 内容 |
|--------------------|------|
| 1. タイムスタンプ付与（受領側） | 取引情報の受領後、速やかにタイムスタンプを付与する |
| 2. タイムスタンプ付与（授受側） | 取引情報の授受時に、タイムスタンプが付されたデータを授受する |
| 3. 訂正削除の防止（システム） | 訂正又は削除を行った場合に、その事実及び内容を確認できるシステム、又は訂正削除ができないシステムを利用する |
| 4. 訂正削除の防止（事務処理規程） | 訂正及び削除の防止に関する事務処理規程を定め、備え付け、当該規程に沿った運用を行う |

### 検索要件

| 要件 | 内容 |
|------|------|
| 検索項目 | **取引年月日**、**取引金額**、**取引先** の3項目で検索できること |
| 範囲指定 | 日付又は金額の範囲を指定して検索できること |
| 組合せ検索 | 2以上の項目を組み合わせて検索できること |

### 検索要件の緩和措置

| 条件 | 緩和内容 |
|------|---------|
| 基準期間の売上高が**5,000万円以下** | 検索要件の全てが不要（税務調査時にデータのダウンロードに応じることが条件） |
| 出力書類を取引年月日その他の日付及び取引先ごとに整理して提示・提出できる場合 | 同上 |

### 実務的な対応方法

#### 方法1: ファイル名による管理（最もシンプル）

```
ファイル名の例:
20260315_110000_株式会社デジタルツール研究所.pdf
（取引日_金額_取引先名.拡張子）
```

フォルダ構成例:
```
電子取引データ/
├── 2026年/
│   ├── 01月/
│   │   ├── 20260115_055000_株式会社ABC.pdf
│   │   └── 20260120_033000_DEF商事.pdf
│   ├── 02月/
│   └── 03月/
```

#### 方法2: 索引簿（Excel）による管理

| No. | 取引年月日 | 取引金額 | 取引先 | ファイル名 |
|-----|-----------|---------|--------|-----------|
| 1 | 2026/01/15 | 55,000 | 株式会社ABC | invoice_001.pdf |
| 2 | 2026/01/20 | 33,000 | DEF商事 | invoice_002.pdf |

#### 方法3: 専用システムによる管理

クラウド会計ソフト（freee、マネーフォワード等）やファイル管理システムの検索機能を利用。

### 事務処理規程のサンプル項目

事務処理規程（措置4を選択する場合）には以下の項目を含める:

1. 目的
2. 適用範囲
3. 管理責任者
4. 電子取引データの取扱い
   - 受領・保存方法
   - ファイル名の命名規則
   - 保存場所
5. 訂正及び削除の原則禁止
6. やむを得ず訂正・削除する場合の手続き
   - 申請→承認のフロー
   - 訂正・削除の記録保存
7. 保存期間（法人: 7年、欠損金がある場合10年）

---

## 保存期間

| 区分 | 保存期間 |
|------|---------|
| 法人の帳簿書類 | 事業年度の確定申告書の提出期限の翌日から**7年間** |
| 法人の帳簿書類（欠損金あり） | **10年間**（2018年4月1日以後開始事業年度） |
| 個人事業主の帳簿 | 確定申告期限の翌日から**7年間**（一部5年） |
| 電子取引データ | 帳簿書類と同じ保存期間 |

---

## 罰則

| 違反内容 | 罰則 |
|---------|------|
| 電子取引データを電子保存しなかった場合 | 青色申告の承認取消しの対象となり得る |
| 隠蔽・仮装があった場合 | 重加算税が10%加重（スキャナ保存・電子取引について、データの改ざん等があった場合） |
| 電子帳簿の要件を満たさない場合 | 帳簿書類の保存がなかったものとされ、青色申告の承認取消し、仕入税額控除の否認等のリスク |

## CRMでの実装推奨事項

1. **顧問先の電子取引対応状況の管理**: 対応済み/未対応/対応中のステータス管理
2. **基準期間売上高の管理**: 5,000万円以下かどうかで検索要件の緩和判定
3. **事務処理規程のテンプレート提供**: 顧問先向けに規程テンプレートを用意
4. **タスクリマインド**: 電子帳簿保存に関する定期的なチェックリマインド
5. **研修・説明の記録**: 顧問先への説明実施日・内容の記録
