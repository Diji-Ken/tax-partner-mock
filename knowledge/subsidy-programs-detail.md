# 補助金申請実務ガイド

## 1. GビズIDの取得方法

### 1-1. GビズIDとは
- 複数の行政サービスにログインできる法人・個人事業主向けの共通認証アカウント
- 補助金申請（jGrants）、社会保険手続き（e-Gov）等で使用

### 1-2. アカウントの種類

| 種類 | 特徴 | 申請方法 | 発行期間 |
|------|------|---------|---------|
| **gBizIDプライム** | 法人代表者・個人事業主用。全サービス利用可能。補助金申請に必要 | 書類審査（印鑑証明書等の郵送） or オンライン（マイナンバーカード使用） | 書類: 約2週間、オンライン: 即日〜数日 |
| **gBizIDメンバー** | プライムアカウントの組織内メンバー用。プライムの管理下で利用 | プライムアカウント保有者が発行 | 即日 |
| **gBizIDエントリー** | 簡易版。利用できるサービスが限定的。補助金申請には使えない場合が多い | オンラインで即時発行 | 即日 |

### 1-3. gBizIDプライムの取得手順

#### 書類申請の場合
1. GビズIDのWebサイト（https://gbiz-id.go.jp）にアクセス
2. 「gBizIDプライム作成」を選択
3. 必要事項を入力（法人名、代表者名、メールアドレス等）
4. 申請書をダウンロード・印刷
5. 申請書に実印を押印
6. 以下の書類をGビズID運用センターに郵送:
   - 申請書（実印押印済み）
   - 印鑑証明書（発行後3ヶ月以内の原本）
7. 審査完了後、メールでログインURL・パスワード設定案内が届く

#### オンライン申請の場合（マイナンバーカード使用）
1. GビズIDのWebサイトにアクセス
2. 「gBizIDプライム作成（オンライン）」を選択
3. マイナンバーカードをICカードリーダー or スマートフォンで読み取り
4. 必要事項を入力
5. 審査完了後、利用開始

## 2. jGrants（電子申請システム）の使い方

### 2-1. jGrantsとは
- 経済産業省が運営する補助金の電子申請システム
- GビズIDプライムでログインして利用

### 2-2. 利用手順
1. **ログイン**: GビズIDプライムでjGrants（https://www.jgrants-portal.go.jp）にログイン
2. **補助金の検索**: 公募中の補助金を検索・一覧表示
3. **申請フォームの入力**: 事業計画、経費明細等を入力
4. **添付書類のアップロード**: 必要書類をPDF等でアップロード
5. **申請の送信**: 入力内容を確認して送信
6. **審査状況の確認**: マイページで進捗を確認
7. **交付決定後の手続き**: 実績報告もjGrantsから提出

### 2-3. 注意点
- 公募締切直前はアクセス集中でシステムが重くなることがある
- 添付ファイルのサイズ制限に注意（補助金により異なる）
- 申請後の修正は原則不可（差戻しの場合のみ修正可能）

## 3. IT導入補助金の申請フロー

### 3-1. 制度概要（2024年度の例）
| 項目 | 通常枠 | インボイス枠（インボイス対応類型） | セキュリティ対策推進枠 |
|------|--------|-------------------------------|-------------------|
| 補助率 | **1/2** | **3/4〜4/5**（小規模事業者） | **1/2** |
| 補助額 | 5万円〜450万円 | 〜350万円 | 5万円〜100万円 |
| 対象 | ITツール全般 | 会計・受発注・決済ソフト | セキュリティサービス |

### 3-2. IT導入支援事業者への登録
- IT導入補助金を利用するには、**IT導入支援事業者**（ベンダー/サービス事業者）と連携する必要がある
- IT導入支援事業者が登録しているITツールのみが補助対象
- 申請は**IT導入支援事業者と共同**で行う

### 3-3. 交付申請の手順
1. **IT導入支援事業者の選定**: 導入したいITツールを提供する事業者を選ぶ
2. **GビズIDプライムの取得**: 事前に取得しておく
3. **SECURITY ACTION宣言**: IPA（情報処理推進機構）のSECURITY ACTION制度で「一つ星」以上を宣言
4. **みらデジ経営チェック**: 独立行政法人中小企業基盤整備機構の「みらデジ」で経営チェックを実施
5. **申請情報の入力**: IT導入支援事業者と協力して、jGrantsから申請
6. **事業計画の作成**: 労働生産性の向上率等のKPIを設定
7. **申請の送信**

### 3-4. ITツールの選定
- IT導入補助金事務局のWebサイトで登録済みITツールを検索可能
- ソフトウェア、クラウドサービス、ハードウェア（PC・タブレット等）が対象
- ハードウェアはインボイス枠のみ対象（PC: 上限10万円、タブレット: 上限10万円、レジ: 上限20万円）

### 3-5. 事業実績報告
- 補助事業完了後、実績報告をjGrantsから提出
- ITツールの導入・支払い完了を証明する書類を添付
  - 契約書・注文書の写し
  - 請求書・領収書の写し
  - 振込明細・通帳の写し
  - 導入したITツールの画面キャプチャ等
- 報告期限内に提出しないと補助金が受けられない

## 4. ものづくり補助金の事業計画書の書き方

### 4-1. 制度概要
| 項目 | 内容 |
|------|------|
| 正式名称 | ものづくり・商業・サービス生産性向上促進補助金 |
| 補助率 | 1/2（小規模事業者は2/3） |
| 補助上限 | 750万円〜5,000万円（申請枠・従業員数による） |
| 対象 | 革新的な製品・サービスの開発、生産プロセスの改善 |

### 4-2. 審査ポイント（公募要領に基づく）

| 審査項目 | 配点の目安 | 記載すべき内容 |
|---------|-----------|-------------|
| **技術面** | 高 | 革新性（新規性、独自性）、課題解決力、技術的実現可能性 |
| **事業化面** | 高 | 市場ニーズ、競争優位性、事業化のスケジュール、販路開拓 |
| **政策面** | 中 | 地域経済への貢献、雇用創出、生産性向上 |
| **補助事業の適格性** | 必須 | 賃上げ要件、付加価値額増加、給与支給総額増加 |

### 4-3. 加点項目
| 加点項目 | 内容 |
|---------|------|
| 経営革新計画の承認 | 都道府県から承認を受けている |
| 事業継続力強化計画の認定 | BCP策定済み |
| 賃上げ加点 | 給与支給総額の増加率が高い |
| 被災事業者 | 災害による被災歴がある |
| デジタル技術の活用 | DX推進に資する取組 |

### 4-4. 事業計画書の構成（推奨）
1. **会社概要・事業内容**（1ページ）
2. **現状の課題**（1〜2ページ）: 具体的なデータ・数値で示す
3. **補助事業の内容**（3〜5ページ）: 革新性を明確に、図表を活用
4. **事業化計画**（2〜3ページ）: 市場規模、ターゲット、販路
5. **数値目標**（1ページ）: 付加価値額、給与支給総額の3〜5年計画
6. **スケジュール**（1ページ）: ガントチャート形式が望ましい

## 5. 小規模事業者持続化補助金の計画書の書き方

### 5-1. 制度概要
| 項目 | 内容 |
|------|------|
| 補助率 | **2/3** |
| 補助上限 | **50万円**（賃金引上げ枠等は200万円） |
| 対象 | 小規模事業者（商業・サービス業: 常時5人以下、その他: 20人以下） |
| 対象経費 | 機械装置等費、広報費、ウェブサイト関連費、展示会等出展費、旅費、開発費、委託・外注費等 |

### 5-2. 経営計画書の記載項目
1. **企業概要**: 事業内容、沿革、従業員数、売上高の推移
2. **顧客ニーズと市場の動向**: 自社を取り巻く環境、競合分析
3. **自社や自社の提供する商品・サービスの強み**: SWOT分析的な視点
4. **経営方針・目標と今後のプラン**: 具体的な数値目標

### 5-3. 補助事業計画書の記載項目
1. **補助事業で行う事業名**: 30文字以内で端的に
2. **販路開拓等の取組内容**: 具体的な活動計画
3. **業務効率化（生産性向上）の取組内容**: 該当する場合
4. **補助事業の効果**: 売上増加、新規顧客獲得等の具体的な数値目標

## 6. 採択後のフロー

### 6-1. 全体の流れ
```
公募申請
  ↓
採択通知
  ↓
交付申請 → 交付決定
  ↓
補助事業の実施（事業実施期間内に完了）
  ↓
実績報告の提出
  ↓
確定検査（書面審査・現地調査）
  ↓
補助金額の確定通知
  ↓
補助金の請求
  ↓
補助金の入金（精算払い）
```

### 6-2. 重要な注意点
- **交付決定前の経費は補助対象外**: 必ず交付決定後に事業を開始する
- **事業実施期間内に完了**: 期間内に発注・納品・支払いの全てを完了させる
- **計画変更の場合は事前承認**: 内容や経費の変更は事前に変更申請が必要
- **概算払い**: 一部の補助金では事業完了前に概算払いが受けられる場合がある

### 6-3. 事業実施のルール
| ルール | 内容 |
|--------|------|
| 発注方法 | 補助金の額に応じて相見積もりが必要（一般的に50万円以上で2社以上） |
| 支払方法 | 銀行振込が原則（現金払いは認められない場合が多い） |
| 証拠書類 | 見積書、発注書、納品書、請求書、振込明細を全て保存 |
| 財産管理 | 取得した財産は補助金の目的に沿って使用（処分制限期間あり） |

## 7. 補助金の会計処理

### 7-1. 圧縮記帳（直接減額方式）

補助金で取得した固定資産の取得価額から補助金相当額を減額し、将来の減価償却費を減少させることで、補助金に対する課税を繰り延べる。

#### 仕訳例: 機械装置1,000万円を購入、補助金500万円を受領

**補助金受領時**:
```
借方: 普通預金 5,000,000    貸方: 国庫補助金受贈益 5,000,000（特別利益）
```

**機械装置取得時**:
```
借方: 機械装置 10,000,000   貸方: 普通預金 10,000,000
```

**圧縮記帳（直接減額方式）**:
```
借方: 固定資産圧縮損 5,000,000（特別損失）  貸方: 機械装置 5,000,000
```

- 圧縮後の帳簿価額: 10,000,000 - 5,000,000 = **5,000,000円**
- 以後、5,000,000円を基に減価償却

### 7-2. 圧縮記帳（積立金方式）

#### 仕訳例: 同上

**補助金受領時**:
```
借方: 普通預金 5,000,000    貸方: 国庫補助金受贈益 5,000,000
```

**機械装置取得時**:
```
借方: 機械装置 10,000,000   貸方: 普通預金 10,000,000
```

**圧縮積立金の積立**:
```
借方: 繰越利益剰余金 5,000,000  貸方: 圧縮積立金 5,000,000（純資産の部）
```

**減価償却時（毎年）**:
- 帳簿価額10,000,000円を基に減価償却を計算
- 圧縮積立金の取崩し: 圧縮額 ÷ 耐用年数分を毎年取崩し

```
借方: 圧縮積立金 XXX  貸方: 繰越利益剰余金 XXX
```

**注意**: 法人税の申告調整が必要（別表四で減算、別表五で積立金の管理）

### 7-3. 国庫補助金等特別勘定
- 補助金の交付決定後、固定資産の取得前に決算期末を迎えた場合に使用
- 特別勘定を設定して補助金相当額を繰り延べる

**仕訳**:
```
借方: 国庫補助金等特別勘定繰入額 5,000,000  貸方: 国庫補助金等特別勘定 5,000,000
```

- 翌期に固定資産を取得した時点で特別勘定を取り崩し、圧縮記帳を適用

### 7-4. 消費税の取扱い

#### 補助金は不課税
- 補助金の受領自体は消費税の課税対象外（不課税取引）
- 補助金に消費税は含まれない

#### 仕入税額控除の調整
- 補助金で取得した資産に係る消費税の仕入税額控除は通常通り適用可能
- ただし、消費税の確定申告で補助金に係る仕入控除税額の返還が必要な場合がある

**返還が必要なケース**:
- 補助金の交付要綱で「消費税仕入控除税額の報告・返還」が定められている場合
- 免税事業者・簡易課税の場合は返還不要（仕入税額控除を受けていないため）

**実務上の対応**:
1. 補助金の消費税に係る仕入控除税額を計算
2. 補助事業完了後の確定申告で仕入控除税額を確定
3. 補助金事務局に報告
4. 返還額が確定したら返還

## 8. 不正受給のリスクと罰則

### 8-1. 不正受給の例
- 虚偽の申請内容（売上の過少申告、従業員数の虚偽申告等）
- 架空の経費の計上
- 補助対象外の経費を補助対象として申請
- 関連会社間の取引で不当に補助金を受給
- 補助事業を実施していないのに実績報告を提出

### 8-2. 罰則
| 措置 | 内容 |
|------|------|
| **補助金の返還** | 交付された補助金の全額返還 |
| **加算金** | 返還額に対して年10.95%の加算金 |
| **延滞金** | 返還期限を過ぎた場合に年10.95%の延滞金 |
| **公表** | 企業名・不正内容の公表 |
| **刑事罰** | 補助金等適正化法違反: 5年以下の懲役又は100万円以下の罰金 |
| **今後の応募資格の喪失** | 一定期間、補助金への応募ができなくなる |

## 9. 補助金の併用ルール

### 9-1. 基本ルール
- 同一の事業・経費に対して複数の補助金を重複して受給することは原則**不可**
- 異なる事業・経費であれば、複数の補助金を併用することは**可能**

### 9-2. 併用可能な例
| 組み合わせ | 可否 | 備考 |
|-----------|------|------|
| IT導入補助金（ソフトウェア）+ ものづくり補助金（機械装置） | **可能** | 対象経費が異なる場合 |
| ものづくり補助金 + 小規模事業者持続化補助金 | **可能** | 対象経費が異なる場合 |
| 同じ設備にIT導入補助金 + ものづくり補助金 | **不可** | 同一経費の重複 |

### 9-3. 補助金と税制優遇の併用
| 組み合わせ | 可否 | 備考 |
|-----------|------|------|
| 補助金 + 中小企業経営強化税制 | **可能** | 補助金控除後の自己負担分について税制優遇を適用 |
| 補助金 + 圧縮記帳 | **可能** | 圧縮記帳は補助金受領時の課税繰延措置 |
| 補助金 + 固定資産税の軽減（先端設備等導入計画） | **可能** | 固定資産税の特例措置 |

### 9-4. 実務上の注意
- 申請時に「他の補助金との併用の有無」を記載する欄がある場合が多い
- 併用の可否は各補助金の公募要領で確認すること
- 国の補助金と自治体の補助金の併用は認められることが多い（ただし公募要領で確認）
- 経費の切り分けを明確にし、証拠書類を分けて管理すること
