# e-Tax連携

## 公式ドキュメント
- e-Tax公式サイト: https://www.e-tax.nta.go.jp/
- ソフトウェア開発者向け: https://www.e-tax.nta.go.jp/shiyo/index.htm
- 仕様書ダウンロード: https://www.e-tax.nta.go.jp/shiyo/index.htm（各種仕様書をWord/Excel形式で提供）
- クラウドサービス連携API: デジタル庁公開「クラウドサービス等を利用した法定調書の提出を行うAPI仕様書」

## 重要: e-TaxはREST APIを公開していない

e-Taxは一般的なREST APIを公開しておらず、以下の連携方式を採用しています。

### 連携方式一覧

| 方式 | 概要 | 対象 |
|------|------|------|
| e-Taxソフト(WEB版) | ブラウザベースの操作画面 | 一般ユーザー・税理士 |
| e-Taxソフト(DL版) | Windows専用デスクトップアプリ | 税理士・法人 |
| クラウドサービス連携API | 認定クラウドサービスからの直接送信 | 認定クラウドサービス事業者のみ |
| 会計ソフト連携 | 各会計ソフトからXBRL/CSVエクスポート | 会計ソフトユーザー |

### クラウドサービス連携API（制限あり）
- デジタル庁が共通インターフェース仕様を公開
- **認定クラウドサービス事業者のみ**利用可能
- 一般の税理士事務所が直接APIを呼ぶことはできない
- 認定を受けた会計ソフト（freee、MF等）経由で利用

## XBRL形式の申告データ

### 対応形式
- **XBRL 2.1 Spec**: e-Tax開始時（2004年）からの標準形式
- **CSV形式**: 2020年4月以降、財務諸表データのCSV形式提出に対応

### XBRLタクソノミ
- 国税庁が公開するe-Tax用タクソノミに準拠
- タクソノミは国税庁サイトからダウンロード可能
- 法人税、所得税、消費税それぞれ別タクソノミ

### 作成フロー
1. 会計ソフトで決算データを作成
2. XBRL形式（または CSV形式）にエクスポート
3. e-Taxソフトで電子署名を付与
4. e-Tax受付システムに送信

## 受付番号の管理

e-Taxでの申告送信後に「受付番号」が発行されます。

| 項目 | 説明 |
|------|------|
| 受付番号 | 申告ごとにユニークな番号 |
| 受付日時 | 送信完了日時 |
| 利用者識別番号 | 16桁の固有番号 |
| 確認方法 | e-Taxメッセージボックスで確認 |

## 実装における注意事項

### 税理士向けシステムでの活用
1. **XBRLテンプレート生成**: 会計データからXBRL形式の申告書データを生成
2. **受付番号データベース**: 申告済みの受付番号を管理し、顧問先ごとに追跡
3. **e-Taxソフト連携**: 生成したXBRLファイルをe-Taxソフトにインポート

### 制約事項
- 電子署名にはマイナンバーカードまたは税理士証明書が必要
- e-Taxソフト(WEB版)はWindows + IE/Edgeのみ正式サポート（一部Macにも対応）
- クラウドサービス連携APIの利用には認定事業者としての申請が必要
- XBRL作成には国税庁のタクソノミに精通している必要がある
